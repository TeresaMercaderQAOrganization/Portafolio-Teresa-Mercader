Proyects

Pagina Web – QA Manual y Automation

I perfomance quality assurance on a fully developed gardening and plant-care web application, working collaboratively within anagil Scrum framework. Work management was carried out through weekly sprints using Jira, and I actively participed in Sprint Planning, dailys and Sprint Review at the end of each sprint.

I designed test cases for user stories following the BDD, metodology, based of the defined acceptance criteria. I created Gherkin based scenaris to ensure readable and undestandable tests for the entery team, managing them with AssertThat to maintain traceability between requirements,tests and results with Jira.Tests were exsecuted per user story, and results were according accross both web and mobile enviroments.

Throught each speach I conducted functional testing and, al the end of the sprint, exploratorty testing to identify defects no covered by predefined cases. During the process, I also participed in a Scrum level 8,one of the most complex in the proyect focused on validation the product filters within application. All detected defects were reported in Jira, and linked to their corresponding user stories to esure proper traceability between requirements tets, test and issues.

At the end of each sprint, I prepared the release in Jira and document all delivery relatded information in Confluence included full test plan, execution, results, and the products quality status.

After the release, I identified the application main workfloss and autoamating the regresion Tests ´ utilizando using Playwright and Pytest with Python. Theses tests were integrated into a continuos integration piperline with GitHub Actions, configured to automatically run the regression suite weekly, alined with the end of each sprint.

Portafolio QA – Desarrolladora Web y QA en Testing de APIs

I developed a web portafolio using HTML5, CSS3 and JavaScript, integrated a form conected to an API built with Supabase. I designed the data base tables, and executed the SQL queries to insert and retrieve test data from the form.

I performed API testing with Postman, creating tests for GET (retrieve), POST (created), PATCH (update) y DELETE (remove). The test suite included validations for positive, negative and authentication,response codes as well as checks for the correct structure.

Additional, I implemented a continous integration workflow using GitHub Actions, configuring Newman para automatically run the Postman test collection in every repository update.
