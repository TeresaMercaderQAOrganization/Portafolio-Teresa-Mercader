Proyects

Pagina Web – QA Manual y Automation

I perfomance quality assurance on a fully developed gardening and plant-care web application, working collaboratively within anagil Scrum framework. Work management was carried out through weekly sprints using Jira, and I actively participed in Sprint Planning, dailys and Sprint Review at the end of each sprint.

I designed test cases for user stories following the BDD, metodology, based of the defined acceptance criteria. I created Gherkin based scenaris to ensure readable and undestandable tests for the entery team, managing them with AssertThat to maintain traceability between requirements,tests and results with Jira.Tests were exsecuted per user story, and results were according accross both web and mobile enviroments.

Throught each speach I conducted pruebas funcionales y, al final del mismo, pruebas exploratorias para identificar defectos no cubiertos por los casos definidos. En este proceso participe tambien en un Scrum de dificultad 8,uno de los más complejos del proyecto, centrado en la validadción de los filtros de productos de la aplicación.Los errores detectados se reportaron en Jira, vinculándolos a la historia de usuario correspondiente para asegurar la correcta trazabilidad entre requisitos, pruebas y defectos.

At the end of each sprint, I prepared the release in Jira and document all delivery relatded information in Confluence included .full test plan, execution, results, and the products quality status.

Tras la release, se identificaron los flujos principales de la aplicación y se automatizaron las pruebas de regresión utilizando Playwright y Pytest con Python. Estas pruebas se integraron en un flujo de integración continua con GitHub Actions, configurado para ejecutar automáticamente la regresión de forma semanal, alineada con el cierre de cada sprint.

Portafolio QA – Desarrolladora Web y QA en Testing de APIs

Desarrollé un portafolio web utilizando HTML5, CSS3 y JavaScript, integrando un formulario conectado a una API generada con Supabase. Se diseñaron las tablas de la base de datos y se realizaron consultas en SQL para la inserción y consulta de datos de prueba del formulario.

Realicé testing de APIs utilizando Postman, creando pruebas para las peticiones GET (consulta), POST (creación), PATCH (edición) y DELETE (eliminación). Se implementaron tests que validan códigos de respuesta positivos, negativos y de autenticación, así como la correcta estructura y contenido de las respuestas.

Además, se implementó un flujo de integración continua con GitHub Actions, configurando Newman para ejecutar automáticamente las pruebas creadas en Postman en cada cambio del repositorio.
